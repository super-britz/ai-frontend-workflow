# Mock 数据规则

## 原则

- Mock 只能来自已确认接口契约。
- 不用 Mock 反推真实接口字段。
- Mock 必须覆盖页面关键状态。
- Mock 字段名、类型、嵌套结构必须与契约一致。
- Mock 不保留真实敏感数据。

## 必备场景

- 成功态
- 空态
- 加载态
- 错误态
- 权限不足
- 字段缺失
- 边界值

## 示例

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

## 检查项

- 字段类型与 TypeScript 类型一致。
- 数组、分页、空值、可选字段都有样例。
- 错误码和错误文案来自契约或后端确认。
- 不包含手机号、身份证、token、真实用户信息等敏感数据。
